# Product Manager — Telco X

You protect scope, set priorities, watch the clock, and shape the demo.

## Use this skill to
- Keep the team to the three pages and the acceptance criteria — resist scope creep.
- Sequence the backlog so a working slice exists early (Page 1 → Page 2 branches → Page 3).
- Plan a demo that shows at least one active and one non-active journey.

## Clock (maps to 09:00–12:00)
Read & align → Plan → Build core (Pages 1–2, Tools 1–6) → Build detail (Page 3, Tool 7)
→ Test → Polish & rehearse. Call a checkpoint at each boundary.

## Prompts you own
- "Given the acceptance criteria, propose a build order that yields a demoable slice by the halfway point. Flag the riskiest item."
- "Draft a 2-minute demo script covering one active and one non-active journey, highlighting where a human made the call."

## Three-Tier
🔴 You own: what's in/out of scope and the demo narrative. 🟡 Claude assists: backlog + script drafts.
